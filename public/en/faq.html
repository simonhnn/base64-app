What is Base64? FAQ and guide

The mechanics, uses, and caveats of Base64, explained clearly for beginners.

Base64 basics

Base64 is an encoding scheme that represents binary data — such as images or arbitrary byte sequences — using only 64 characters: letters (A–Z / a–z), digits (0–9), and the symbols + and /, plus = for padding. Think of it as a shared conversion rule that lets data pass through text-only channels or storage formats without being corrupted.

The mechanism is simple: the original data is split into 3-byte (24-bit) groups, regrouped into four 6-bit chunks, and each 6-bit chunk maps to one character. For example, encoding "Hello" yields "SGVsbG8=".

Common uses

  • Email attachments (MIME): to send binary attachments as text.
  • Data URIs: to embed small images or fonts directly in HTML/CSS.
  • Basic authentication: to encode a username and password in an HTTP header.
  • Embedding in JSON / XML: to include binary within structured text.
  • JWT (tokens): each part is represented in URL-safe Base64.

Frequently asked questions (FAQ)

Is Base64 encryption?

No. Base64 is a reversible conversion that anyone can undo, with no secret key. Encoding a password or personal information with Base64 does not protect it, so combine it with encryption such as AES when confidentiality is required.

How much does the size increase?

Since 3 bytes are represented by 4 characters, the data size grows by roughly 1.33x (about 33% larger). Line breaks and padding add a little more.

What is the "=" at the end?

= is padding, added to align the output to multiples of 4 characters when the original length is not a multiple of 3. It is a padding marker, not the data itself.

How does it differ from URL-safe Base64?

URL-safe Base64 replaces + and / — which have special meaning in URLs and file names — with - and _. It is used in JWTs and similar cases.

Why does decoding fail?

It fails when the input does not follow the Base64 format (4-character units, allowed characters, padding), or when the result is an invalid UTF-8 byte sequence. Check for stray whitespace or missing characters.

Is my input sent anywhere?

The conversion is done entirely in your browser, and data is not sent externally for conversion. See the privacy policy for details.
